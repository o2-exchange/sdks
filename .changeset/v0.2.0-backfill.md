### Breaking changes
- Renamed `DepthSnapshot.buys`/`.sells` to `.bids`/`.asks` (all SDKs)
- Changed `Trade.timestamp` from `str` to `int` (Python), `string` to `number` (TypeScript)

### Features
- Added `trader_side` field to Trade model — exposes maker/taker/both role from account-scoped trade endpoints (all SDKs)
- Ported on-chain revert code decoding to TypeScript (was only in Python and Rust)
- Added `stream_depth` wire precision fix — all precision levels 1-18 now support live delta streaming (all SDKs)
- Added input validation at API boundaries: `Id()` (Python), `hexId()` (TS), `IntoValidId` (Rust)
- Added account-filtered `get_trades` via `/v1/trades_by_account` (all SDKs)
- Added pagination with cursor support on `get_trades` and `get_orders` (all SDKs)

### Fixes
- Fixed `augment_revert_reason` to return clean decoded error name instead of multi-KB receipt dumps (all SDKs)
- Fixed `get_bars()` documentation — timestamps are in milliseconds, not seconds (all SDKs)
- Truncated raw revert reasons to 200 chars when no `Revert(DIGITS)` pattern is found
- Fixed Rust `augment_revert_reason` to return clean decoded string (was appending to original reason)
